## TL;DR

The single-domain skill is now a **multi-domain, no-code AI call-center platform**:
config-as-data bundles, a client-side graph flow engine, a React no-code admin
builder, prebuilt industry templates, and an enforced compliance surface. PRD
numbered roadmap **P0–P5 is complete**; only **P6 (live handoff / multi-tenancy)**
is deferred. Branch `dev`, all work pushed to `origin/dev`.

Hard constraints (do not violate): CPU/WASM, minimal infra, static-host friendly;
**no new SDK runtime deps** (admin/ has its own deps and is decoupled); immutable
(never mutate inputs); no `eval`; functions <=50 lines; no emoji; no `console.log`
in shipped `src/`.

## Architecture map

Runtime SDK (`src/`, vanilla ESM, framework-agnostic):
- `src/engine/flow-engine.js` — `createFlowEngine({bundle})`. Graph mode active iff
  `bundle.flow.nodes.length > 0`; otherwise the legacy intent->scenario path is
  byte-identical (back-compat). Node kinds: start, message, intent-branch, slot-fill,
  action, ai-disclosure, handoff, end (http-call = deferred no-op). MAX_HOPS=50 loop
  guard; one-shot per-turn intent latch; from_entity via `sendUserText(text,{entities})`.
- `src/engine/condition.js` — `evaluateCondition(expr, ctx)`. Eval-free, prototype-
  pollution-safe, sealed 13-operator allowlist, `{"var":"slot.x"}` refs. Exported.
- `src/intent/keyword-resolver.js` — no-download default IntentResolver.
- `src/intent/embedding-resolver.js` — opt-in `createEmbeddingIntentResolver` (G3, built).
  Same `prepare()`/`classify()` seam; MAX cosine over EN+KO prototype utterances; default
  `Xenova/multilingual-e5-small` int8 with auto `query:` prefix; Transformers.js loaded ONLY
  via a lazy dynamic `import()` in the default loader (keyword default stays no-download).
  Inject via `createFlowEngine({intentResolver})`. zeroshot/setfit tiers share the seam (not built).
- `src/core/create-call-center.js` — overlay + adapters; consumes engine `{text, actions}`.
- `src/engine/local-rule-engine.js` — preserved legacy engine (do not remove).

Admin builder (`admin/`, own package.json — React 19 + @xyflow/react 12 + ajv, vite 8):
- `admin/src/lib/` — framework-free, node-testable: `flow-bundle.js` (bundleToFlow/
  flowToBundle; strips RF runtime fields; condition string->object on export; hydrates
  node.type from data.type on import), `validate-bundle.js` (Ajv 2020, both schemas by
  $id), `node-kinds.js` (registry mirroring engine reads).
- `admin/src/components/` — thin React UI (canvas/palette/inspector/metadata/toolbar).

Config + templates:
- `schemas/config-bundle.schema.json` (v2) `$ref`s `schemas/call-scenario.schema.json`.
- `bundles/{support,finance,education,insurance}.bundle.json` — support is generated
  from `scenarios/*.yml` via `scripts/build-bundle.mjs --check`; the 3 templates are
  hand-authored. All carry governance + bilingual disclosure (enforced by tests).

## Invariants to preserve

- Empty `flow` MUST keep the legacy intent->scenario behavior byte-identical.
- `support.bundle.json` is generated — do NOT hand-edit it; change `scenarios/*.yml`
  then `npm run scenarios:build` + `npm run bundle:build` (validated by `--check`).
- The admin only consumes the public schema + (in tests) `src/api.js`; it must not edit
  or tightly couple to SDK internals. SDK keeps zero new runtime deps.
- Every shipped bundle must keep a complete `governance` block + bilingual `disclosure`
  (enforced by `tests/compliance.test.mjs`).
- Condition expressions: no eval, no prototype access; unknown operator throws.

## Known limitations / deferred (low severity, documented)

- intent-branch `data.fallback` is editable in the admin as EN string only; the engine
  reads `{en,ko}` objects too (KO branch fallback not exposed in the UI).
- Admin canvas node summary text is not truncated (cosmetic; React-text-safe).
- Templates use 7 of 8 node kinds (no slot-fill node); declared `topic` from_intent
  slot mappings are inert in graph mode (only fill via a slot-fill node).
- Pure-message cycles run to MAX_HOPS and emit a degenerate payload (authoring error;
  belongs to admin build-time cycle validation).
